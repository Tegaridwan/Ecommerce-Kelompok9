Identifikasi Alat Bantu (Tools)

1. Visual Studio Code 
Visual Studio Code sangat krusial dalam pembaruan modul pembayaran berkat fitur IntelliSense yang memberikan saran auto-complete pintar saat mendefinisikan Class maupun Interface, sehingga proses penulisan kode menjadi jauh lebih cepat dan akurat. Sebagai mahasiswa TRPL yang mengutamakan efisiensi, fitur bawaan seperti Error Highlighting sangat membantu karena otomatis menyoroti kesalahan sintaks (biasanya dengan garis bergelombang merah) bahkan sebelum kode dijalankan. Selain itu, fitur built-in debugger memungkinkan pemasangan breakpoint untuk menghentikan eksekusi program sementara, sehingga alur logika perhitungan pembayaran dan status variabel dapat ditelusuri baris demi baris untuk melacak titik error secara real-time.

2. Git SCM 
Penggunaan Git SCM mutlak diperlukan saat merombak arsitektur krusial seperti modul pembayaran karena sistem ini bertindak sebagai rekam jejak digital sekaligus jaring pengaman (safety net). Perombakan pada fitur transaksi memiliki risiko kegagalan yang tinggi; dengan Git, fitur baru dapat dikembangkan dan diuji secara terisolasi menggunakan sistem Branch tanpa takut merusak kode utama (main branch) yang sudah berjalan stabil. Jika pembaruan yang baru saja di-commit ternyata memicu bug sistem atau kesalahan logika fatal, Git memungkinkan proses rollback untuk mengembalikan source code secara instan persis ke versi stabil sebelumnya.

3. Aplikasi UML (Draw.io)
Untuk memvisualisasikan arsitektur sistem, alat bantu yang digunakan adalah Draw.io, sebuah platform diagram berbasis web yang praktis dan menyediakan pustaka shape lengkap khusus untuk standarisasi UML (Unified Modeling Language). Penggunaan alat ini sangat membantu pemahaman desain karena mampu menerjemahkan logika Object-Oriented Programming yang abstrak menjadi rancangan Class Diagram yang konkret. Dengan menggambar rancangan di Draw.io, relasi antar entitas—seperti letak atribut private, method spesifik, serta garis hubung untuk Inheritance (pewarisan) dan implementasi Interface—dapat dilihat secara helikopter, sehingga kesalahan struktur desain bisa dicegah sebelum proses pengetikan kode dimulai.
